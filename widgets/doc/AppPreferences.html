Using Preferences with Advanced Application Superclasses

Preferences

Programmatically implemented applications that use the wt.apps.BaseApp superclass may include additional app-specific preferences. Preferences are useful to store user-specific settings that will persist between uses of the application.

Programming Interface

The wt.apps.BaseApp superclass provides the following methods for your concrete app class to utilize preferences. While you can view preferences directly from the app's Preferences property, it is recommended to use the interface below to store or retrieve saved preferences in your code. Te below methods will handle the default value when the preference has not been initialized. You should utilize the setPreference method any time the value is updated. Preference values are saved automatically when the app is closed normally.
Name
Access
Description
getPreference
public, sealed
value = getPreference(app,propName,defaultValue)
Retrieves a preference from the Preferences object given a property name and a default value. (The default value is returned if the property does not exist.)
setPreference
public, sealed
setPreference(app,propName,value)
Sets or updates a preference into the Preferences object given a property name and value.
